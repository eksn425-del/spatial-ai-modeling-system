# Codex Role and Limits

Codex is useful in this system because many spatial modeling tasks can be made explicit through data, code, schemas, and documentation.

## Codex is strong at

- Reading and restructuring text-based data.
- Migrating parcel IDs, rules, and notes into JSON.
- Creating schemas and validation scripts.
- Generating first-pass GH Python cells.
- Keeping input and output names consistent.
- Adding try/except, warnings, and summary JSON.
- Writing documentation and GitHub packaging material.

## Codex is limited at

- Reliably judging complex Rhino/GH visual results.
- Knowing whether a tower skin looks spatially coherent without viewport feedback.
- Guaranteeing that facade lines, flowing surfaces, podiums, and roofs are visually acceptable.
- Replacing manual Rhino/GH preview and human design judgment.

## Practical rule

Use Codex for structure, code, data, and documentation. Use Rhino/GH and human validation for spatial quality. Use ChatGPT when the task requires building style breakdown, generator logic, visual debugging language, or GH connection reasoning.
