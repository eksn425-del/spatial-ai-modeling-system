# Human-in-the-loop Principle

The system treats AI output as a structured draft. A model is not successful because code runs. A model is successful only after the human designer verifies data, geometry, preview behavior, visual quality, and documentation accuracy.

## Core principle

AI can accelerate spatial modeling, but it needs clear data structures, modular GH cells, validation mechanisms, and human feedback loops.

## Human checkpoints

- Data checkpoint: parcel IDs, boundaries, roads, and coordinate notes are correct.
- Rule checkpoint: hierarchy, height, mobility, open space, and landscape logic match design intent.
- GH checkpoint: output ports match Python variables and each output previews separately.
- Visual checkpoint: geometry is coherent in Rhino/GH.
- Bake checkpoint: only accepted outputs are baked.
- Story checkpoint: GitHub and portfolio claims are honest.

## Why this matters

AI often produces plausible code that fails visually. Complex building facades, flowing tower skins, and detailed roof/podium relationships need judgment that cannot be replaced by a text-only code pass.
