# 中文产品定位说明｜Spatial AI Modeling System

## 推荐定位

推荐标题：

> Spatial AI Modeling System｜基于 Skills 与 Human-in-the-loop Workflow 的空间建模自动化系统

这个项目最适合包装成 **AI Workflow / Skills / Vibe Coding / AI Coding 工具系统**。

## 用户与场景

- 城市设计 / 建筑设计学生：需要把复杂空间建模流程拆成可复用步骤。
- Rhino / Grasshopper 用户：需要稳定的 GH Python 输入输出结构和验证机制。
- AI 建模学习者：需要理解 AI 适合做什么、不适合做什么。
- 空间产品设计师：需要从数据、规则和意图快速生成可验证原型。

## 用户痛点

- 空间建模流程经常混在一个大 GH 电池组里，难以 debug。
- AI 生成复杂建筑表皮或城市体块时，容易出现输出缺失、summary null、Preview/Bake 混乱。
- 数据、规则、代码和视觉判断之间缺少标准流程。
- 项目完成后很难转化成可解释的 GitHub 产品案例。

## AI 产品价值

```text
site data + parcel rules + design intent
-> JSON rule system
-> Codex-assisted GH Python cell generation
-> Rhino/GH preview
-> summary JSON + validators
-> human validation
-> GitHub / portfolio packaging
```

AI 的价值不是自动完成设计，而是把混乱的空间建模任务拆成可验证、可复用、可讲清楚的 Workflow。

## 项目叙述建议

这个项目体现的是我把 AI 辅助空间建模经验产品化的能力。我没有把它包装成自动建筑生成器，而是把 AI 建模拆成 data、rules、GH cells、generators、validation、prompts 和 packaging。它更接近 AI Workflow 产品：Codex 做结构化代码和文档，人类在 Rhino/GH 中验证几何与视觉质量。
