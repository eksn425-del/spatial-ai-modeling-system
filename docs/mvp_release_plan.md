# MVP Release Plan

## v0.1 goal

Make the repository understandable and demonstrable as a GitHub MVP. A visitor should quickly understand what the system is, how to start from sample data, how to enter the GH Python cell workflow, how validation works, and what demo materials still need to be added.

## Must have for v0.1

- README
- QUICK_START
- SYSTEM_MANIFEST
- 10 skills
- schemas
- templates
- validators
- minimal example
- screenshot placeholders

## Already included

- Product README and manifests
- Workflow map
- 10 skill documents
- JSON schemas
- GH Python and building generator templates
- Standard-library validators
- Minimal fictional site example
- Single tower generator structure example
- Prompt library
- Checklists
- Portfolio summary drafts
- Assets README with screenshot guidance
- Demo script

## Release checklist

- Confirm validators run on sample data.
- Confirm README has Current Status.
- Confirm QUICK_START can be followed by a beginner.
- Confirm assets folder explains what can and cannot be uploaded.
- Add screenshots or placeholders before sharing publicly.
- Keep all claims human-in-the-loop and avoid saying AI automatically designs buildings.
