# Interview Talking Points

## 30-second Version

Spatial AI Modeling System is my Agent-assisted urban modeling workflow. I built it to convert messy urban-design tasks into structured inputs, JSON rules, modular GH Python cells, Rhino/GH validation steps, and portfolio-ready documentation. The point is not that AI automatically designs the city, but that AI helps make the modeling process repeatable, debuggable, and explainable.

## 60-second Version

In the Maluan TOD workflow, I treated AI as a modeling workflow partner. I broke the task into site data, parcel IDs, height rules, building groups, GH Python components, Rhino/GH previews, and validation reports. Codex helped structure rules, generate first-pass GH Python cells, write validators, and package the workflow. But the final spatial judgment still happens in Rhino/GH: checking parcel counts, height compliance, preview/bake states, skyline logic, and whether the model supports the design intent. This project shows my ability to design an Agent-assisted workflow, not just use AI as a code shortcut.

## How to Answer "Did AI Design the Project?"

No. AI did not replace design judgment. I used AI to structure and accelerate the modeling workflow: data organization, rule migration, component scaffolding, validation scripts, and documentation. Human review remained necessary for spatial hierarchy, visual quality, and model acceptance.

- Do not say the workflow replaces Rhino/GH expertise.
- Do not say the system proves design quality.
