# GitHub Product Packaging

## Purpose

Package the spatial AI modeling workflow as a clear GitHub product instead of a hidden personal modeling backup.

## When to Use

Use this skill after the core system, templates, validators, prompts, and examples exist, or when preparing the project for portfolio and internship use.

## Inputs

- Product README
- Manifests
- Skill docs
- Schemas
- Templates
- Validators
- Example data
- Screenshots or demo captures
- Human validation notes

## Outputs

- GitHub-ready README
- Product positioning docs
- Case study template
- Resume bullets
- Application statement
- Release checklist

## Procedure

1. Present the project as Spatial AI Modeling System, not as an architecture portfolio backup.
2. Explain the workflow in product terms: data, rules, GH cells, generators, validation, prompts, and packaging.
3. Include clear limitations: not a fully autonomous architecture generator.
4. Show human-in-the-loop validation as a strength.
5. Add screenshots or demo media only after Rhino/GH validation exists.
6. Avoid claiming that AI automatically designs buildings.
7. Describe Codex and ChatGPT roles accurately.

## Codex Role

Codex writes documentation, organizes folders, creates manifests, maintains schemas, and prepares release checklists.

## ChatGPT Role

ChatGPT helps refine portfolio storytelling, application language, and case-study framing.

## Human Validation

The human designer selects demo images, confirms claims are accurate, and decides what can be publicly shown.

## Common Failures

- The project reads like a traditional architecture portfolio rather than a workflow product.
- The README overclaims autonomy.
- Screenshots are missing, so the workflow has no visual proof.
- Failure cases are hidden instead of used as product lessons.

## Recovery Strategy

- Reframe the project around AI product thinking and design automation workflow.
- Add a "What this system is not" section.
- Add failure cases and lessons.
- Add demo screenshots, Rhino/GH captures, and short workflow clips later.
- Keep GitHub language honest: AI-assisted, validated, human-in-the-loop.

## Deliverables

- GitHub README
- Product manifest
- System manifest
- Roadmap
- Portfolio one-page
- Resume bullets
- Release checklist
