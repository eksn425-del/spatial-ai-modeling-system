# Roadmap

## Phase 0: Project skeleton

- Create product manifests.
- Define skill folders.
- Add schemas, templates, validators, prompts, checklists, and examples.
- Establish the human-in-the-loop principle.

## Phase 1: Minimal reproducible workflow

- Validate sample site and parcel data.
- Generate one simple GH Python cell from JSON.
- Preview site boundary, roads, parcel outlines, and basic massing.
- Output panel summary and summary JSON.

## Phase 2: Rule-driven city massing

- Add parcel hierarchy rules.
- Add height rule validation.
- Generate separated outputs for towers, podiums, open space, roads, labels, and warnings.
- Avoid one giant GH component.

## Phase 3: Building generator pattern

- Develop single-building generator templates.
- Separate glass core, facade lines, podium breps, roof breps, and all_breps.
- Test style parameters through Rhino/GH previews.
- Document common failures and recovery strategies.

## Phase 4: Product packaging

- Add screenshots and demo captures.
- Write a case study using the product case study template.
- Prepare resume bullets and application statement.
- Publish to GitHub as a workflow system.

## Phase 5: Future productization

- Web UI for editing JSON rules.
- Visual parameter editor.
- Automatic GH cell generator.
- Rendering export.
- Model QA dashboard.
- Case-study builder for portfolio release.
